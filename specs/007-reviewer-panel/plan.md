# Implementation Plan: Reviewer Panel

**Branch**: `007-reviewer-panel` | **Date**: 2026-03-20 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/007-reviewer-panel/spec.md`

## Summary

Extend the existing reviewer panel with document upload capability and fix the task rejection validation bug. The reviewer can now upload documents directly from the application detail page, with a `source_type` column distinguishing reviewer uploads from client and admin uploads. A mandatory rejection reason is enforced. A `<x-reviewer-layout>` component brings visual consistency with the admin and client panels. All new routes are protected by a new `documents.reviewer-upload` permission.

## Technical Context

**Language/Version**: PHP 8.2+ / Laravel 11
**Primary Dependencies**: spatie/laravel-permission v6+, Laravel Blade, local/S3 filesystem (via `FILESYSTEM_DISK`)
**Storage**: MySQL (MAMP local dev); SQLite in-memory (tests); `private` local disk for files
**Testing**: PHPUnit / Laravel feature tests (`php artisan test`)
**Target Platform**: Laravel web application (MAMP local dev → production server)
**Project Type**: Web application (Blade SSR, MVC)
**Performance Goals**: Dashboard queue loads in under 2 seconds for up to 100 active applications
**Constraints**: All file uploads validated via Form Request; storage driver swappable via `.env`
**Scale/Scope**: Reviewer queue ~10–50 concurrent applications; no pagination required this phase

## Implementation Phases

### Phase 1: Database & Model Layer

**Goal**: Add `source_type` column, update model and service, add permission.

**Tasks**:

1. Create migration `add_source_type_to_documents_table`: add `$table->string('source_type', 20)->default('client')->after('uploaded_by')`.
2. Update `Document::$fillable` to include `'source_type'`.
3. Update `DocumentService::upload()` signature: add `string $sourceType = 'client'` parameter; pass to `Document::create()`.
4. Update all existing callers of `DocumentService::upload()` to pass explicit source type:
   - `App\Http\Controllers\Client\DocumentController@store` → pass `'client'`
   - `App\Http\Controllers\Admin\DocumentController@store` → pass `'admin'`
5. Add `documents.reviewer-upload` to permissions list in:
   - `database/migrations/2026_03_20_000001_seed_roles_permissions_and_visa_types.php`
   - `database/seeders/RolePermissionSeeder.php`
   Assign to `reviewer` role in both files (using `syncPermissions`).

### Phase 2: Authorization Layer

**Goal**: Add reviewer upload policy method.

**Tasks**:

6. Add `reviewerUpload(User $user): bool` to `DocumentPolicy` → returns `$user->can('documents.reviewer-upload')`.

### Phase 3: Bug Fix — Rejection Reason Required

**Goal**: Enforce non-empty rejection reason (FR-005, US2 Scenario 5).

**Tasks**:

7. Update `RejectTaskRequest::rules()`: change `note` from `['nullable', 'string', 'max:2000']` to `['required', 'string', 'min:5', 'max:2000']`.
8. Update `resources/lang/en/reviewer.php`:
   - Change `note_label` → `'Rejection Reason'`
   - Add `reject_note_placeholder` → `'Describe why this task is being rejected (required)...'`
   - Add `reject_note_required` → `'A rejection reason is required.'`
9. Update `resources/lang/ar/reviewer.php` with corresponding Arabic translations.
10. Update `resources/views/reviewer/applications/show.blade.php`: change rejection form label/placeholder to use new keys; mark field as required.

### Phase 4: Reviewer Document Upload

**Goal**: Allow reviewers to upload documents from the application detail page (FR-010, FR-011, US4).

**Tasks**:

11. Create `app/Http/Requests/Reviewer/UploadDocumentRequest.php`:
    ```
    rules: file (required|file|mimes:pdf,jpg,jpeg,png|max:10240), application_task_id (required|integer|exists:application_tasks,id)
    ```
12. Create `app/Http/Controllers/Reviewer/DocumentController.php` with `store()` method:
    - Authorize with `$this->authorize('reviewerUpload', Document::class)`
    - Retrieve `$task = ApplicationTask::findOrFail(...)`, abort_if task doesn't belong to application
    - Call `$this->documentService->upload($application, $task, $request->file('file'), Auth::user(), 'reviewer')`
    - Redirect to `reviewer.applications.show` with success flash
13. Register route in `routes/web.php`:
    ```php
    Route::post('/applications/{application}/documents', [ReviewerDocumentController::class, 'store'])
        ->middleware('can:documents.reviewer-upload')
        ->name('applications.documents.store');
    ```
    (inside the existing `reviewer.` prefix group)
14. Add upload lang keys to `resources/lang/en/reviewer.php`:
    - `upload_document` → `'Upload Document'`
    - `upload_select_task` → `'Select task'`
    - `upload_success` → `'Document uploaded successfully.'`
    - `upload_section_title` → `'Upload Document'`
15. Add corresponding Arabic keys to `resources/lang/ar/reviewer.php`.
